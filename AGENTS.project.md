# AGENTS.project.md

> 作用域：仓库根目录及其全部子目录。
> 本文件只保留日志治理执行硬规则；完整 rationale 见 `docs/logging/README.md`。

## 执行硬规则

- 修改阶段流程、worker 流程、外部资源边界时，必须同步检查日志布点。
- 新增 `except ...: continue`、fallback、重试、自动回退、降级路径时，必须同时补日志。
- 新增线程、后台任务、异步加载、桥接回调时，必须覆盖启动、完成、失败三个边界。
- 不能把承担阶段可见性的业务日志随意下沉到 `DEBUG/TRACE`。
- 不能只为当前报错补一条日志，必须补全该逻辑点应有的最小日志集合。
- 第三方桥接默认只做等级映射、来源标签和最小上下文，不得改写成业务阶段事件。
- 默认优先直接使用 `logger.xxx(...)`。
- 业务层先决定布点和等级，再决定是否需要薄封装。

## 薄封装准入规则

只有同时满足以下条件，才允许新增薄 logging API：

- 同一种日志模式在 3 处及以上重复出现。
- 新 API 能减少重复样板或阶段文案漂移。
- 不破坏 caller attribution，必要时显式处理 caller depth。
- 不削弱 `loguru` 原生能力。
- 业务层调用写法不比直接 `logger.xxx(...)` 更重。

任一条件不满足时，继续直接使用 `logger.xxx(...)`。
